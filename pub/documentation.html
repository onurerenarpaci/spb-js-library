Getting Started

Download the CSS and JS files

segmented_progress_bar.js
segmented_progress_bar.css

Import the source files and JQuery

<link rel="stylesheet" href="segmented_progress_bar.css" />

<script defer src="https://ajax.googleapis.com/ajax/libs/jquery/3.5.1/jquery.min.js" ></script>

<script defer type="text/javascript" src="segmented_progress_bar.js" ></script>

Create a div for the Segmented Progress Bar in your html body. Give it an id and give the class "spb"

<div id="example_spb" class="spb"></div>

Initiliaze Segmented Progres Bar in your code

// Create the config object and set the preffered options
const spb_config = {
    length: 500,
    horizontal: true,
    segmentHoverColor: '#1ae300',
}

// Create the spb object by giving the div id and config object as arguments
const spb1 = spb('#example_spb', spb_config);

// Add segments and give their titles and percentages as arguments
spb1.addSegment('Uploading the setup files are to the server', 12);
spb1.addSegment('Extracting files', 15);
spb1.addSegment('Installing Dependencies', 18);
spb1.addSegment('Install the Ipsum software', 30);
spb1.addSegment('Delete setup files', 15);
spb1.addSegment('Run the server', 10);

API Documentation

spb(selector, config)

Main library function, takes a selector string for selecting the spb div, and takes a config object for layout and styling options. Returns a spb object

.addSegment(title, percentage)

Crates a segment for the spb. Takes a title string and a percentage number. When finished adding segments sum of percentages should be equal to 100. Created segments have index numbers in the order they have been created

.updateStatus(index, percentage)

Updates the status of a particular segment. Takes the index of the segment and the percentage that will be updated as arguments

.updateDescription(index, title)

Updates the description of a particular segment. Takes the index of the segment and the title that will be updated as arguments. Generally used when the segment is finished.

.log(index, percentage)

Used for inserting a single line of log to a particular segment. Takes the index of the segment and the log string that will be pushed to the log list.

Config options

width

Type: String. Width of the spb in pixels

height

Type: Number. length of the spb. Used for segment partititon, so it needs to be number

horizontal

Type: Boolean. If true, the horizontal layout will be used, vertical otherwise.

backgroundColor

Type: String. css compatible color code of the spb background

logBackgroundColor

Type: String. css compatible color code of the log window background

segmentBackgroundColor

Type: String. css compatible color code of the process segment background

statusColor

Type: String. css compatible color code of the process status

segmentHoverColor

Type: String. css compatible color code of the process segment border when hovered with mouse

totalStatusColor

Type: String. css compatible color code of the total status percentage
